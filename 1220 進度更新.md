## 🆕 3. 整合 Google Cluster Trace 真實資料集

### 為什麼需要 Google Cluster Trace？

PlanetLab 非常適合做學術基準實驗，但仍有以下限制：

- 僅提供 CPU 使用率，**沒有真實 PM 容量**
- VM / PM 規模偏小（~1000 VM）
- 無法完整呈現真實大型資料中心的異質性

因此本專案額外整合 **Google Cluster Trace (2011)**，用來：

- 補足 **真實資料中心規模**
- 使用 **真實 PM CPU / Memory 容量**
- 驗證演算法在 **工業級工作負載** 下的表現
- 強化專案的實務與論文等級完整性

---

### 資料集來源

- 官方資料集：Google Cluster Trace (2011)
- 廣泛用於 VM placement / scheduling / energy-aware cloud 研究
- 原始系統：Google Borg Scheduler

本研究使用以下檔案：

- `task_usage/part-00000-of-00500.csv.gz`
- `machine_events/machine_events.csv.gz`

（僅取子集以控制資料規模）
