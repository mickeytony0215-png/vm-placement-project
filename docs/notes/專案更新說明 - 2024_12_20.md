### **1\. 整合 Google Cluster Trace 真實資料集**

**為什麼需要 Google Cluster Trace？**

* PlanetLab 僅提供 CPU 使用率，**沒有真實 PM 容量**。  
* Google Trace 包含真實的 12,583 台機器的 CPU 與 Memory 規格。  
* 驗證演算法在 **工業級、異質性工作負載** 下的表現。  
* 強化專案實務價值，測試處理資源不平均（CPU vs RAM 比例不一）的能力。

**資料來源與處理：**

* 官方資料集：Google Cluster Data (2011) / Borg 系統追蹤數據。  
* 使用檔案：task\_usage (VM 需求) 與 machine\_events (PM 容量)。  
* 工具：開發 src/utils/google\_trace\_to\_dataset.py 自動化預處理。

**我們使用的資料（取樣）：**

* **VM 規模**：10,000 個 VM（從 17 萬個候選任務中隨機抽取）。  
* **PM 規模**：2,000 台 PM（從 1.2 萬台真實機器中隨機抽取）。  
* **取樣邏輯**：採固定 RANDOM\_SEED=42，保留工作負載的長尾分佈特性。

### ---

**2\. 數據處理與計算邏輯**

本研究將原始 Trace 的標準化比例值轉換為統一資源尺度，以利 VM placement 運算。

**VM 資源需求計算：**

* **資料來源**：task\_usage.csv  
* **CPU 需求**：cpu\_demand \= max(cpu\_rate) × 100  
* **Memory 需求**：memory\_demand \= max(max\_memory\_usage) × 100  
* **備註**：採峰值策略，確保靜態放置方案能應付最大負載。canonical\_memory 為請求值，故本研究採用 max\_memory\_usage 以反映真實緊湊度。

**PM（Physical Machine）容量計算：**

* **資料來源**：machine\_events.csv  
* **PM CPU capacity**：cpu\_capacity \= raw\_cpu\_capacity × 100  
* **PM memory capacity**：memory\_capacity \= raw\_memory\_capacity × 100  
* **特性**：包含多種異質配置（例如：50.0/24.93、25.0/24.98 等）。

**3\. Google Trace 與 PlanetLab 的角色區分**

| 項目 | PlanetLab | Google Trace (GCT) |
| :---- | :---- | :---- |

### ---

**4\. 實驗結果初步觀察 (實驗 4：Google Trace)**

根據 10,000 VM 規模的壓力測試結果：

| 指標 | FFD | BFD | RLS-FFD |
| :---- | :---- | :---- | :---- |
| **活躍 PM 數量** | **836** | 993 | **836** |
| **CPU 利用率** | **92.35%** | 83.52% | **92.35%** |
| **能源消耗** | **223,517** | 269,930 | **223,517** |
| **執行時間** | **0.76s** | 9.38s | 3.46s |

**結果分析：**

* **FFD / RLS-FFD**：表現穩定，不到 1 秒即可找到極佳解，CPU 利用率突破 92%。  
* **BFD**：不適用於異質比例環境。多用了 157 台 PM，且執行時間顯著增加，顯示單純追求「最小殘餘空間」容易導致單一維度資源先耗盡。

**🛠️ 使用方法 (Usage Guide)**

### **1\. src/utils/google\_trace\_to\_dataset.py**

**功能：**

* 從 Google Trace 原始 CSV 壓縮檔產生 2000 PM / 10000 VM JSON 資料集。  
* 依 (job\_id, task\_index) 聚合成單一 VM 數據並採用峰值需求策略。  
* 自動處理 machine\_events 獲取真實異質 PM 容量。

**執行方式：**

Bash

python src/utils/google\_trace\_to\_dataset.py

**輸出：**

* 終端機顯示處理進度與聚合 VM 數量。  
* 儲存 JSON 到 data/google\_dataset/ 目錄。

### **2\. experiments/run\_all\_experiments.py**

**功能：**

* 一鍵執行 Experiment 1 至 4，自動生成 JSON 格式結果報告。  
* 顯示詳細執行過程、PM 數量、能源與 CPU 利用率。

**執行方式：**

Bash

python experiments/run\_all\_experiments.py
